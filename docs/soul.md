# CDUTAgentOS — 产品灵魂

## 一句话

**让每一台 ESP32 都有大脑。**

## 核心价值

CDUTAgentOS 解决一个真实问题：当部署数十甚至数百台 ESP32 传感终端时，如何集中管理设备、采集数据、并让 AI 参与决策。

它不是又一个后台管理系统，而是 **设备操作系统**——管理设备的生命周期（注册、配置、监控、指令），同时桥接 AI 能力到每一台终端。

## 产品架构

```
┌─────────────────────────────────────┐
│            AI 模型引擎               │  ← 推理、对话、决策
├─────────────────────────────────────┤
│          AIoT 管理平台               │  ← 设备管理 + 服务器管理 + 用户管理
├─────────────────────────────────────┤
│         TCP Server 网关              │  ← 设备接入、协议转换
├─────────────────────────────────────┤
│      ESP32 无线传感终端集群           │  ← 数据采集、指令执行
└─────────────────────────────────────┘
```

## 设计取舍

### 选 Tornado 而不是 Django
设备长连接场景需要单进程管理多个 TCP 连接。Tornado 的 IOLoop 天然适合。Django 的 ORM 和中间件在这里是负资产。

### 选 SQLite 而不是 MySQL
小规模部署（< 1000 设备）SQLite 完全够用，零运维成本。单文件数据库备份只需复制一个文件。扩展到大规模时再切换 PostgreSQL。

### 选 JSON + \n 而不是 MQTT
ESP32 资源有限，JSON 文本协议足够轻量。MQTT 需要额外 Broker，增加部署复杂度。当设备量增长到需要消息队列时再引入。

### 选服务端渲染 + LayUI v2.13.7 而不是 SPA
管理后台不需要 React/Vue 的构建链。LayUI v2.13.7 完整组件包（表格、表单、弹层、导航、布局）覆盖后台全部 UI 场景。服务端渲染 + LayUI 增强，首屏快、零构建、开发效率高。

## 安全原则

- 密码 PBKDF2-SHA256 慢哈希，100k 次迭代，16 字节随机 salt
- XSRF 防护所有表单提交
- Secure Cookie 管理登录态
- 生产部署前更换 `cookie_secret`
